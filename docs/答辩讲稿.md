# 每日打卡习惯追踪APP答辩讲稿

## 一、开场介绍

各位老师好，我的作品名称是《每日打卡习惯追踪APP》。本项目是基于 Android Studio Panda 2 环境开发的一款安卓应用，主要用于帮助用户记录日常习惯，并通过每日打卡的方式培养自律性。

## 二、项目目标

本项目主要完成了以下几个核心功能：

1. 登录功能，并支持记住密码；
2. 习惯事件的新增、删除、修改；
3. 展示事件标题、内容、提醒时间和创建时间；
4. 支持调用摄像头拍照和从相册选择图片；
5. 页面整体采用较简洁美观的交互设计。

## 三、演示流程

### 1. 登录演示

首先打开应用，可以看到登录页面。

这里我可以输入账号和密码，勾选“记住密码”后登录。为了便于演示，系统预置了一个账号：

- 用户名：`student`
- 密码：`123456`

如果没有账号，也可以点击右上角的“注册账号”进行注册。

### 2. 首页演示

登录成功后进入主界面。上方会显示：

1. 当前登录用户名；
2. 今日概览；
3. 习惯总数；
4. 今日已打卡数；
5. 最长连续打卡天数。

### 3. 新增习惯

点击右下角新增按钮，可以输入：

1. 事件标题；
2. 事件内容；
3. 提醒时间；
4. 拍照或从相册添加图片。

保存之后，该习惯就会显示在列表中。

### 4. 编辑与删除

每个习惯卡片下方都有“编辑”和“删除”按钮。

1. 编辑可修改标题、内容、时间和图片；
2. 删除可移除当前习惯。

### 5. 打卡功能

点击卡片右上角的打卡按钮，可以完成今日打卡。

打卡后：

1. 该习惯状态点会变成绿色；
2. 今日打卡统计会增加；
3. 连续打卡天数会自动更新。

### 6. 图片功能

在新增或编辑时，可以调用手机摄像头拍照，也可以从系统相册选择图片。图片会显示在习惯卡片中，方便记录学习笔记、运动截图或生活记录。

## 四、技术实现说明

本项目主要采用以下技术：

1. Kotlin 语言；
2. Jetpack Compose 构建页面；
3. ViewModel + StateFlow 进行状态管理；
4. DataStore 保存登录信息；
5. JSON 文件保存账号和习惯数据；
6. FileProvider、Activity Result API 实现拍照与图片选择。

## 五、项目亮点

我认为本项目有以下几个亮点：

1. 功能完整，覆盖了课程要求；
2. 不依赖网络，便于离线演示；
3. 支持图片记录，展示效果更直观；
4. 页面风格统一，操作流程清晰。
