# 每日打卡习惯追踪APP作品报告

## 一、课题名称

每日打卡习惯追踪APP设计与开发

## 二、项目背景

随着移动互联网的发展，越来越多的人希望借助手机应用培养自律习惯，例如晨跑、背单词、阅读、健身等。传统纸质记录方式不方便统计，也不便于长期坚持。因此，本项目设计并开发了一款轻量级“每日打卡习惯追踪APP”，帮助用户进行事件管理、每日打卡记录与图片留存。

## 三、项目目标

本项目围绕课程要求完成以下目标：

1. 实现登录功能，并提供记住密码选项；
2. 实现习惯事件的新增、删除、修改等功能；
3. 展示每个事件的标题、内容、时间等信息；
4. 支持调用摄像头拍照、从相册选择图片；
5. 页面美观，交互流畅，满足演示与答辩需求。

## 四、开发环境

- 开发工具：Android Studio Panda 2 | 2025.3.2
- 开发语言：Kotlin
- UI 框架：Jetpack Compose
- 构建工具：Gradle Kotlin DSL
- 最低支持版本：Android 8.0（API 26）
- 目标版本：Android 14（API 34）

## 五、系统功能设计

### 1. 登录模块

用户输入用户名和密码后可以进入系统，勾选“记住密码”后，下次打开应用会自动保留账号密码信息。系统内置演示账号，也支持新用户注册。

### 2. 习惯管理模块

用户进入主界面后可以查看自己的习惯列表。每条习惯都显示：

1. 事件标题；
2. 事件内容；
3. 提醒时间；
4. 创建时间；
5. 连续打卡天数；
6. 附加图片。

同时支持新增、编辑、删除。

### 3. 每日打卡模块

用户点击打卡按钮后，可将对应习惯标记为今日已完成。系统会自动统计：

1. 今日已打卡数量；
2. 当前连续打卡天数；
3. 习惯总数。

### 4. 图片采集模块

在新增或编辑习惯时，用户可以：

1. 直接调用手机摄像头拍照；
2. 从系统相册中选择图片；
3. 将图片保存到应用私有目录中并展示在习惯卡片中。

## 六、系统实现方案

### 1. 数据存储

本项目采用本地离线存储方式：

1. `DataStore Preferences` 用于保存登录状态、记住密码信息；
2. JSON 文件用于保存账号数据和习惯数据；
3. 应用私有目录用于保存拍照或相册复制后的图片资源。

这种方案结构清晰、实现难度适中，适合课程项目展示，也不依赖网络环境。

### 2. 页面设计

项目主要包含以下页面或区域：

1. 登录页；
2. 注册弹窗；
3. 主界面概览卡片；
4. 习惯列表卡片；
5. 新增/编辑习惯底部弹层。

整体采用 Material 3 风格，配色简洁柔和，交互逻辑清楚，适合演示。

### 3. 关键技术点

1. 使用 `Jetpack Compose` 进行声明式 UI 开发；
2. 使用 `rememberLauncherForActivityResult` 调起相机和相册；
3. 通过 `FileProvider` 安全分享应用私有目录中的拍照文件；
4. 使用 `ViewModel + StateFlow` 管理界面状态；
5. 使用日期集合计算连续打卡天数。
